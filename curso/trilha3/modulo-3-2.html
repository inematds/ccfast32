🔁 /loop — Monitoramento Automático

Ficar verificando manualmente se um deployment foi concluído, se um PR está aprovado ou se logs de erro apareceram é trabalho desperdiçado. /loop delega esse monitoramento ao Claude — ele reexecuta um prompt em intervalos definidos e só te chama quando há algo que precisa da sua atenção.

🔁 Como o /loop Funciona

O Claude reexecuta o prompt especificado dentro da mesma sessão no intervalo definido. A sessão permanece ativa e o Claude executa a verificação automaticamente até uma condição ser atingida ou até você interrompê-lo.

  • • Loops duram até 3 dias consecutivos
  • • Intervalo definido em linguagem natural
  • • Interrompe automaticamente quando algo precisa de atenção
  • • Lembretes únicos também funcionam (sem repetição)
Claude Code — Exemplos de /loop

# Monitorar deployment

Você: A cada 5 minutos, verifique se o deployment do PR #234 completou.

Se falhar, me avise com o log de erro.

[Claude verifica a cada 5 min]

Claude: Verificando... deployment ainda em andamento (12 min).

Claude: Verificando... deployment ainda em andamento (17 min).

Claude: ⚠️ ATENÇÃO: Deployment falhou. Erro: OOMKilled no pod api-server.

Log completo: [...]

# Lembrete único

Você: Às 15h30 me lembre de fazer o deploy da feature de pagamentos.

Claude: Anotado. Vou te lembrar às 15h30.

# Monitorar erros em produção

Você: A cada 10 minutos, verifique os logs do Sentry por novos erros

críticos. Pare quando não houver novos erros por 30 minutos.

💡 Dica Prática

Defina a condição de parada explicitamente: "pare quando o deployment estiver verde" ou "pare quando não houver novos erros por 30 minutos". Sem condição de parada, o Claude continua até o limite de 3 dias. Para monitoramento de produção, sempre inclua o que fazer quando o problema for encontrado.

🖥️ VPS — Sessões Sempre Ativas

Tarefas longas têm um inimigo: o laptop fechado. Quando a tela apaga, a sessão morre, o trabalho para. Rodar o Claude Code em um VPS resolve isso completamente — a sessão continua no servidor remoto, independente do que acontece com sua máquina local.

🖥️ Arquitetura VPS + Claude Code

O Claude Code roda no servidor remoto como qualquer processo de terminal. O tmux ou screen mantém a sessão viva mesmo depois de você fechar o SSH. Para controle avançado, conecte a um bot do Telegram.

  • • VPS (Ubuntu 22.04+) com pelo menos 2 vCPUs e 4GB RAM
  • • tmux mantém sessões após desconectar SSH
  • • Bot do Telegram para enviar comandos via chat
  • • Código do projeto fica no servidor — não na máquina local
Terminal — Setup VPS com tmux

# 1. Conectar ao VPS

$ ssh user@seu-vps.com

# 2. Criar sessão tmux persistente

$ tmux new-session -s claude-work

# 3. Iniciar Claude Code no projeto

$ cd ~/projeto && claude

> Claude Code iniciado. Sessão ativa.

# 4. Desconectar sem matar a sessão

$ Ctrl+B, D # Detach do tmux

$ exit # Fechar SSH — Claude continua rodando

# 5. Reconectar mais tarde

$ ssh user@seu-vps.com

$ tmux attach -t claude-work

> [Claude ainda trabalhando na tarefa]

⚠️ Atenção — Segurança

Quando o Claude Code roda em VPS, o código do projeto fica no servidor remoto. Garanta que o VPS tenha acesso restrito (chaves SSH apenas, sem senha), firewall configurado e backups regulares. Nunca armazene credenciais em plain text no servidor.

💡 Dica Prática — Telegram Bot

Configure um bot do Telegram que envia comandos ao Claude via stdin e retorna o output via Telegram. Você fica com controle total pelo celular sem precisar abrir SSH. Perfeito para verificar status rapidamente durante o dia sem precisar abrir o terminal.

📱 Controle Remoto pelo Celular

Diferente do VPS (onde o código fica no servidor), o controle remoto nativo do Claude Code mantém o código na sua máquina local mas permite controlar a sessão de qualquer dispositivo — incluindo o celular. Você começa na mesa, sai para uma reunião, e continua orientando o Claude pelo telefone.

📱 Como Funciona o Controle Remoto

O Claude Code expõe uma interface web local que pode ser acessada remotamente via túnel seguro. O código nunca sai da máquina — apenas os comandos e outputs viajam pela conexão.

1

Inicie no computador

Abra o Claude Code e ative o modo de controle remoto

2

Obtenha o link de acesso

Um URL seguro é gerado para aquela sessão específica

3

Acesse pelo celular

Abra o URL no navegador do celular — interface completa disponível

4

Continue orientando

Envie mensagens, veja o output e guie o Claude à distância
